## Cel

Wybierzesz program, w którym będziesz edytować dokumenty na swoim komputerze.

## Czy edytor jest konieczny

Nie.  
Jeśli pracujesz bezpośrednio na stronie GitHub, nie potrzebujesz dodatkowego edytora.

Edytor jest potrzebny wtedy, gdy pracujesz z GitHub Desktop.

## Zalecany edytor

Rekomendujemy **Visual Studio Code**.

Możesz też używać innego edytora, jeśli już go znasz.

## Skonfiguruj edytor w GitHub Desktop

1. Otwórz GitHub Desktop.
2. Otwórz ustawienia programu. Ścieżka do ustawień może różnić się zależnie od systemu operacyjnego i wersji programu. 
3. Przejdź do sekcji **Integrations**, **Advanced** albo **External editor** — nazwa może się różnić zależnie od wersji programu.
4. Wybierz edytor, którego chcesz używać.
5. Zapisz ustawienia.

W systemie Windows ustawienia zwykle znajdują się w menu „File”, a w macOS — w menu aplikacji GitHub Desktop.


## Jeśli edytora nie ma na liście

Zainstaluj edytor i uruchom ponownie GitHub Desktop.

Jeśli nadal go nie ma, możesz wskazać plik uruchamiający edytor ręcznie.  
W razie problemów poproś o pomoc osobę techniczną.
